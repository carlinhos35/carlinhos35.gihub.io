Imagens dinâmicas

Podemos criar imagens dinâmicas, ou seja, carregar a imagem de acordo com o tamanho requisitado.

Para isso, além da tag <img>, usamos a tag <picture> e a tag:

<source media = "(max-width: 750px)" srcset = "imagens/foto-p.png">

Onde max-width: 750 refere-se a largura máxima da imagem.

Podemos testar aumentando ou diminuindo a largura do navegador, pode tentar.

No caso usado aqui o código ficou da seguinte maneira, nesse caso a ordem do source importa.

    <picture>    
        <source media = "(max-width: 750px)" srcset = "imagens/foto-p.png">
        <source media = "(max-width: 1050px)" srcset = "imagens/foto-m.png">
        <img src="imagens/foto-g.png" alt="Imagem Grande">
    </picture>

Reproduzindo audios

Para inserir um audio usando HTML5 usamos o tag <audio> com o endereço da mídia e o comando "controls".

Exemplo:

<audio src="midias/just-dance.mp3" controls></audio>

Audios "dinâmicos"

Assim como acontece com imagens dinâmicas, podemos fazer algo semelhante com o audio.

Para driblarmos possíveis problemas de compatibilidade de arquivos entre navegadores podemos utilizar a mesma ideia de imagens dinâmicas, o ou seja, colocar uma lista de audios em diferentes formatos de modo que caso não seja possível o navegador carregar o primeiro então ele vai para o segundo e assim por diante.

Os principais arquivos suportados são mp3, ogg e wav, porém arquivos wav são grandes e devem ser evitados.

Para realizar esse "dinamismo" de audio usamos o seguinte código.

            <audio preload = "metadata" controls>    
                <source src = "midias/just-dance.mp3 type = "audio/mpeg">
                <source src = "midias/just-dance.ogg type = "audio/ogg">
                <source src = "midias/just-dance.wav type = "audio/wav>   
                <p>Caso não consiga ouvir o audio click <a href="midias/just-dance.mp3">aqui</a> e faça o download.</p>               
            </audio>

Com esse código teremos um resultado visuali logo abaixo igual ao anterior, porém temos uma maior compatibilidade e proteção a problemas.

Note que só vai aparecer o link para download se os três arquivos anteriores falharem.

Em preload podemos usar os seguintes parâmetros:

<audio preload = "auto" controls> Carrega todo o conteúdo do audio

<audio preload = "metadata" controls> Carrega apenas algunsparâmetros do audio.

<audio preload = "none" controls> Só carrega qualquer parâmetro do audio e o próprio audio se for dado play. Nesse caso só seria carregado o comando de download da música após dar o play.
